# Adapter 0.3.2 Release Closeout

Date: 2026-06-21
Status: released

## Scope

This closeout records the final security, performance, packaging, and release
work completed for `npcink-ai-client-adapter` 0.3.2.

The release keeps Adapter inside its thin OpenClaw channel boundary:

- reads still route through WordPress Abilities API;
- governed writes still route through Governance Core proposal and
  commit-preflight truth;
- final writes still require explicit Adapter execution profiles after Core
  approval;
- Adapter still does not own provider/model runtime, Cloud connector truth,
  workflow runtime, or generic final write execution.

## Security And Performance Changes

Eight feedback items were handled before the release:

1. Signed key-pair scopes were split so final execution routes and
   `commit-preflight` require explicit `npcink.execute` or legacy
   `magick.execute`.
2. Device pairing defaults still grant only `npcink.read`, `npcink.propose`,
   and `npcink.status`; `npcink.execute` must be explicitly requested and
   approved.
3. Public device-pairing poll requests now have a transient-backed rate limit
   keyed by IP plus device-code hash.
4. Approved proposal execution now uses a short per-proposal `add_option()`
   lock to reduce duplicate final-write races.
5. Core capabilities and dependency-contract discovery now use request-local
   memoization plus short transient caching for discovery-only paths.
6. Execution responses now default to public summaries. Raw
   `approval_context`, `results`, and `result` are returned only when the
   caller explicitly requests full detail.
7. Upstream error payloads are bounded and redacted before being exposed by
   Adapter errors.
8. Execution records and cached preflight handoffs now prune by time as well
   as count.

Additional hardening:

- `health` and `help` expose the Core app token source as
  `constant`, `environment`, `option`, or `none`, without exposing the token.
- The WordPress Application Password fallback now requires an explicit admin
  acknowledgement.
- Environments may disable the fallback with
  `NPCINK_OPENCLAW_ADAPTER_DISABLE_APPLICATION_PASSWORD_FALLBACK`.

## Guardrails For Future Work

Future Adapter changes should keep these constraints intact:

- Do not add provider credentials, model routing, prompts, or production model
  execution to Adapter.
- Do not add Adapter-owned Cloud connector routes, Cloud settings, signing
  clients, run/result lookup, or artifact registries.
- Do not add generic final write execution.
- Add final writes only as explicit execution profiles after Core approval and
  Core commit-preflight.
- Keep key-pair `npcink.execute` separate from `npcink.propose`.
- Keep public unauthenticated routes body-bounded and rate-limited.
- Keep public execution and upstream error responses redacted by default.
