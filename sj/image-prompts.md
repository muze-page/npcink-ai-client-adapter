# Image Prompts

## Shared Series Identity

Use a consistent Magick AI plugin-family visual language across future assets:

- Modern open-source developer tooling.
- Clean geometric shapes.
- Deep charcoal or clean off-white base.
- WordPress-like blue accents.
- Green validation accents.
- A small magenta/magic accent used sparingly as the Magick AI family signature.
- No official WordPress logo, MCP logo, AI company logos, or trademarked marks.

## Icon Prompt

```text
Create a square WordPress.org plugin icon for "Magick AI Adapter", one plugin in the Magick AI plugin family.

This specific plugin represents a thin OpenClaw channel adapter for WordPress: one REST entry point that connects OpenClaw to Magick AI Core governance and the WordPress Abilities API. It is not the ability content package, not the governance core, not a cloud connector, not an MCP server, and not a model client.

Visual metaphor: a precise protocol bridge or channel gateway between an external client node and a WordPress control-plane node, with two clean lanes: one lane to Core governance and one lane to Abilities API execution contracts. The bridge should look thin, controlled, and bounded, not like a large cloud platform.

Series identity: modern open-source developer tooling, clean geometric shapes, consistent Magick AI family style, restrained technical design.

Color direction: deep charcoal or ink base, WordPress-like blue accents, green routing/ready accents, small magenta/magic accent used sparingly as the Magick AI family signature.

Style: flat vector, crisp, high contrast, professional, readable at 128x128 and 256x256.

Avoid: robot faces, chat bubbles, cloud product imagery, governance dashboards as the main subject, ability catalog nodes as the main subject, official WordPress logo, MCP logo, AI company logos, tiny unreadable text, fantasy magic wand imagery.
```

## Banner Prompt

```text
Create a WordPress.org plugin banner for "Magick AI Adapter", size 1544x500, with a layout that also works when scaled down to 772x250.

This plugin is part of the Magick AI plugin family. Its role is the OpenClaw channel adapter: one WordPress REST namespace for reading Magick AI Core capability guidance, running approved direct-read abilities through the WordPress Abilities API, creating Core proposals, and orchestrating one user-triggered approve-and-execute path through Core.

It complements the other Magick AI plugins:
- Magick AI Abilities handles ability definitions and ability callbacks.
- Magick AI Core handles governance, approval, preflight, and audit.
- Magick AI Cloud Addon links to cloud services.
This banner should clearly represent Adapter routing, not Abilities, Core, or Cloud.

Visual concept: a clean channel gateway between OpenClaw and WordPress. Show an external client endpoint entering a thin Adapter layer, then branching into two controlled paths: Core governance and Abilities API execution contracts. Emphasize connection, REST routing, bounded handoff, and controlled pass-through.

Text: "Magick AI Adapter"
Optional subtitle: "OpenClaw channel adapter for Core governance and WordPress Abilities"

Tone: precise channel infrastructure, secure connection, controlled routing, calm and professional developer tooling.

Series identity: use a visual language that can extend to Abilities, Core, and Cloud Addon: same geometry, same base palette, different symbolic focus.

Color direction: deep charcoal or clean off-white base, WordPress-like blue, green routing/ready accents, small magenta/magic accent used sparingly.

Composition: strong readable title area, abstract adapter bridge on the opposite side, generous margins, no critical details near edges, balanced enough for LTR and RTL contexts.

Avoid: robots, chatbot UI, cloud dashboards, approval/audit panels as the primary metaphor, ability catalog nodes as the primary metaphor, official WordPress logo, MCP logo, AI company logos, fantasy magic effects, marketing hype, small unreadable code text.
```
